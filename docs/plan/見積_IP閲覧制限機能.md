## 1. 見積の位置づけ

### 1.1 本見積の対象

提案書で推奨した **「会場 Wi‑Fi 出口 IP ＋ 本社固定 IP（または VPN 出口）による閲覧制限」** を、本番環境で運用可能な形で実装・ドキュメント化・検証するための **追加開発費用** です。

### 1.2 既存開発範囲との関係

開発計画 Phase 2 に含まれるセキュリティ（§6）とは **別項目** です。

| 内容 | Phase 2 既存範囲 | 本見積（追加） |
|------|------------------|----------------|
| nginx リバースプロキシ・HTTPS 終端 | ○（基盤構築） | △（IP 制御用の設定追加） |
| Origin / 短期トークン / Rate limit | ○ | — |
| **接続元 IP 許可リスト（場内・本社のみ通過）** | **×** | **○** |
| 閲覧拒否時の案内画面（403） | × | ○ |
| 許可 IP リストの運用手順書 | × | ○ |
| 開催前テスト・リスト登録支援 | × | △（オプション） |

> Receiver の LAN 直結 WebSocket（ピット内ローカル配信）は対象外です。本見積は **クラウド URL 経由のブラウザ閲覧** に限定します。

## 6. お見積り条件・免責

1. **IP アドレスの変動**  
   会場 Wi‑Fi の出口 IP が開催ごとに変わる場合、都度リスト更新（オプション D）が必要です。変動型回線のみの場合、追加コスト・場内閲覧不可リスクを事前にご説明します。

2. **LTE のみの来場者**  
   提案書記載のとおり、キャリア回線のみでは場内・場外の判別ができません。運用案内（Wi‑Fi 接続）は貴社広報・運営の範囲です。

3. **完全な秘匿性**  
   IP 制限は一般的なアクセス制御として有効ですが、VPN・プロキシ等による迂回の可能性はゼロではありません。トークン・Origin 検証（Phase 2 既存）との併用を推奨します。

4. **見積有効期限・範囲変更**  
   要件変更（Cloudflare 必須化、GPS 併用等）があった場合は再見積となります。

## 7. 提案先への説明文（コピー用）

> LiveTiming をサーキット場内および岡山市内本社からのみ閲覧可能とし、場外からの一般アクセスを遮断する **IP 許可リスト方式** の追加開発です。  
> 会場 Wi‑Fi および本社固定 IP（または VPN 出口）をクラウド側に登録し、それ以外の接続はページ・リアルタイムデータともに拒否します。
