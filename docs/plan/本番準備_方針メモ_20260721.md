## 1. 背景

ブラウザ開発者ツールで Network / Sources を見ると、WebSocket URL やロジック・JSON 構造が分かる状態だった。  
「外部から直接サーバーへ繋がれ、データが不正取得される」ことへの懸念があり、現状確認と本番に向けた進め方を整理した。

## 2. 現状コードの確認結果（要約）

### 2.1 懸念と実装状況

| # | 懸念 | 判定 |
|---|------|------|
| 1 | `/ws` が無認証で誰でも接続できる | **開発設定では事実**。`FRONTEND_VIEW_TOKEN` 未設定時は認証オフ（`.env.example` 明示） |
| 2 | 通信が平文 `ws://` | **開発想定**。本番は nginx で WSS 終端 |
| 3 | ポート 4000 の直接露出 | **現状はフロントが `:4000` 直結**。nginx 設定はリポジトリ未整備 |
| 4 | `NEXT_PUBLIC_LIVE_WS` がクライアント JS に埋め込まれる | **事実**。URL 隠蔽は本質対策にならない |
| 5 | JSON が素直（`hello` / `state` / `patch` / `smis`） | **事実**。解析しやすい |
| 6 | レート制限・同時接続上限 | **未実装** |
| 7 | サーバー側バリデーション | `/ingest` はあり。`/ws` は一方向配信のみ。`/api/messages` は**認証なし** |
| 8 | 再接続（指数バックオフ） | 正規ユーザー向けだが、スクレイパーにも同じ利点 |
| 9 | ドライバー名等の個人情報 | 公開レースなら通常問題なし。非公開イベントは要配慮 |

### 2.2 すでに守れている部分

- Receiver → `/ingest` は **Bearer トークン必須**
- `/ws` 用の Origin 検証・静的トークン検証の**骨格**は `server/src/auth.ts` / `broadcast-server.ts` にある（デフォルト無効）
- 短期トークン rotation は **未実装**（README 既知制限）

### 2.3 開発中にブラウザで見えるもの

- `ws://<host>:4000/ws`（または `NEXT_PUBLIC_LIVE_WS`）
- メッセージ形式・`useLiveTiming` の差分適用ロジック
- `/debug` での生 JSON ストリーム
- `GET /api/health`, `GET /api/messages`（認証なし）

**認識の要点**: フロントのソースや Network は「秘密」にできない。守るべきは **直叩き・無認証取得をサーバー側で拒否すること**。

## 6. セキュリティ方針（本番で担保するもの）

### 6.1 できること

| 目標 | 手段 |
|------|------|
| `:4000` を外から見えない | nginx 裏、`HOST=127.0.0.1` |
| 生の `ws://IP:4000/ws` を隠す | 同一オリジン `wss://mola-timing-okayama.com/ws` のみ |
| 無認証の直結を拒否 | `/ws` 短期トークン + Origin 検証 |
| 送信は Receiver のみ | `/ingest` Bearer 必須（既存） |
| ソースマップ漏洩低減 | 本番で `.map` 非公開、minify |
| デバッグ面の閉鎖 | `/debug` 本番無効、`/api/messages` 保護または無効 |

### 6.2 できないこと（誤解しやすい点）

正規ユーザーがページを開いている限り:

- DevTools Network に **`/ws` は見える**（同一サイト通信のため）
- 配信 JS は **ダウンロード可能**（難読化は読みにくくするまで）

→ **「URL・ソースを完全に隠す」ではなく「直叩き・不正取得を拒否する」** が本質。

### 6.3 目指す見え方

```
一般:
  https://mola-timing-okayama.com     → ページ表示
  wss://…/ws（トークンなし）         → 拒否
  http://<IP>:4000                   → 外から到達不可

正規ブラウザ:
  同一ドメインの /ws（トークン付き）で視聴

Receiver:
  wss://mola-timing-okayama.com/ingest + 秘密トークンのみ
```
